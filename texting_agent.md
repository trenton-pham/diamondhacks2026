# Texting Agent Spec

## Mission
Generate high-quality conversational messages to counterpart agents while staying within session limits and privacy guidance. Use topic-specific user profile context retrieved at draft time to produce personalized, authentic messages.

## Topic-Based Context Retrieval (RAG-like)

### Overview
During onboarding, the user's real text message history is processed and sorted into **topic-specific markdown files** — one file per topic. These files are **read-only context** for the Texting Agent. Before generating a draft, the agent determines which topic file(s) are relevant to the current conversation and loads them as grounding input — similar to retrieval-augmented generation but over structured topic files rather than vector embeddings.

The Texting Agent **never writes to these files**. They are populated exclusively during the initial ingestion of the user's chat logs.

### Topic File Categories
Each user has a profile directory containing the following topic files, generated once from their uploaded text history:

| File | Contents |
|---|---|
| `hobbies.md` | Activities, pastimes, sports, creative pursuits |
| `values.md` | Core beliefs, priorities, life philosophy, what matters most |
| `interests.md` | Topics they're curious about, media preferences, intellectual interests |
| `humor.md` | Joke style, sarcasm level, meme preferences, comedic tone |
| `dealbreakers.md` | Hard boundaries, non-negotiables, incompatibilities |
| `communication_style.md` | Texting cadence, emoji usage, verbosity, formality level |
| `lifestyle.md` | Daily routines, work-life balance, social habits, living preferences |
| `goals.md` | Short-term and long-term aspirations, ambitions, direction |

### Ingestion Pipeline (Text Sorter)
Topic files are generated by the **Text Sorter** (`text_sorter.md`) during onboarding:
1. **Upload**: User provides their text message export (e.g., iMessage, WhatsApp, SMS backup).
2. **Parse**: Normalize to a unified message format, filter to user-sent messages only.
3. **Classify**: Each message is classified into one or more topic categories.
4. **Summarize**: Distill classified messages into key signals and patterns per topic — no raw message text is stored.
5. **Privacy review**: The Privacy Agent validates that no blocked or sensitive content persists in the generated files.
6. **Write**: Output topic files to `profiles/{user_id}/`.

### Topic File Format
Each topic file follows this structure:
```markdown
# {Topic Name}
<!-- generated_at: ISO-8601 timestamp -->
<!-- source_message_count: number of messages this was derived from -->

## Key Signals
- Bullet-pointed observations extracted from the user's text history

## Patterns
- Recurring themes or preferences observed across conversations
```

### Retrieval Flow
1. **Intent classification**: From the current turn context (conversation history, evaluator tags, intent_tag), determine the relevant topic(s).
2. **File selection**: Load the matching topic file(s) from the user's profile directory. Select 1-3 files max per turn to stay within token budget.
3. **Context injection**: Append the retrieved topic content to the prompt as grounding context.
4. **Draft generation**: Produce the `TextDraft` informed by both conversation history and the user's real texting data.

### Retrieval Rules
- If a relevant topic file is empty or does not exist, fall back to evaluator context and generic rapport-building.
- Topic files are read-only — agent outputs are never appended back into them.
- Retrieved content counts toward the Texting Agent's input token budget (1400 tokens max).

## Allowed Inputs
- Current session context (`session_id`, `turn_index`, prior safe messages).
- Evaluator decision context and intent tags.
- Rewrite guidance from Privacy Agent.
- Session limits (`max_messages`, `max_privacy_retries`, tone settings).
- **Retrieved topic file(s)** from the user's profile directory (1-3 files per turn).

## Disallowed Behavior
- Do not bypass Privacy Agent.
- Do not mention or reveal blocked sensitive topics.
- Do not generate spam, harassment, or manipulative pressure language.
- Do not exceed hard conversation cap.

## Output Schema (`TextDraft`)
```json
{
  "session_id": "string",
  "turn_index": 12,
  "draft_text": "string",
  "intent_tag": "build_rapport",
  "retrieved_topics": ["hobbies", "humor"]
}
```

Field rules:
- `draft_text` must be plain text safe for privacy review.
- `intent_tag` should be from a controlled set (e.g., `intro`, `ask_question`, `build_rapport`, `handoff_prompt`).
- `retrieved_topics` lists which topic files were used as context for this draft (for auditability).

## Refusal Behavior
- If context is insufficient or hard limits are reached, return no draft and reason:
```json
{
  "session_id": "string",
  "turn_index": 12,
  "draft_text": "",
  "intent_tag": "refuse_limit_or_context"
}
```
